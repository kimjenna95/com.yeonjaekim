Venmo- Case Study

A reimagined Venmo app with a new feature that would help users keep track with monthly payments and requests.

About

Venmo is a mobile app that acts like a digital wallet. It lets you make and request payments with others while also letting you connect with people and remember the moments you share with friends.

Problem

Venmo is a great and easy way to pay and get paid. However, as easy as it may be, it could get very tedious - especially if it’s a payment you have to request every week or month. Users who need to request or pay recurring payments often forget to collect or pay on a recurring basis. It becomes another small task to worry about on top of their busy lives.


Based off my user interviews, I wanted to create a feature that allows Venmo to create scheduled transactions to ease the struggles users had to face. There a few objectives I wanted to cover in creating this new feature.


Objectives:
· Add a feature to create autopay for recurring payments/requests
· Allow users to control payment splitting options with multiple recipients
· Allow users to customize the frequency and time of recurrence

Persona

User Flows

The user flow served as a template on how a user might interact with the new ‘Scheduled Transaction’ feature. The flow assumes the user is already familiar with the current functionalities of Venmo.

Sketches

Lo-fi Wireframes

Lo-fi wireframes served as a baseline on what the product could look like. During the first iteration of usability testing, I was able to take user feedback and make updates for the high fidelity prototype.

User Journey

Usability Testing

Usability testing was done unmoderated through asking participants to create a scheduled transaction without any help. Through reflecting on the behavior patterns and observations, the new feature was a success. The new feature was straightforward and user-intuitive. The option to be able to split the cost amongst friends was a great add-on. In the end, the scheduled transaction feature seemed functional and seamless.
